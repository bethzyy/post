# 2026年AI趋势搜索项目 - 核心架构要求

## 🎯 核心目标

**程序调用不同的AI模型，每个模型自己通过其特有的工具或方式进行实时网络搜索**

## 📋 模型及其搜索方式

### 1. GLM-4.6 (智谱AI)
- **API接口**: Anthropic兼容API
- **端点**: `https://open.bigmodel.cn/api/anthropic`
- **库**: `anthropic`
- **模型名**: `glm-4.6`
- **搜索方式**: 通过API调用，让模型基于其训练数据和搜索能力生成热点

### 2. Claude (Anthropic)
- **API接口**: Anthropic官方API
- **端点**: `https://api.anthropic.com`
- **库**: `anthropic`
- **模型名**: `claude-sonnet-4-5-20250514` 或最新版本
- **搜索方式**: 通过API调用，让模型基于其知识库生成热点

### 3. ChatGPT (OpenAI)
- **API接口**: OpenAI官方API
- **端点**: `https://api.openai.com/v1`
- **库**: `openai`
- **模型名**: `gpt-4` 或 `gpt-4-turbo`
- **搜索方式**: 通过API调用，让模型基于其知识库生成热点

### 4. Gemini (Google)
- **API接口**: Google AI Studio API
- **端点**: `https://generativelanguage.googleapis.com/v1beta`
- **库**: `google.generativeai`
- **模型名**: `gemini-2.0-flash-exp` 或最新版本
- **搜索方式**: 通过API调用，让模型基于其知识库生成热点

### 5. Pollinations.ai
- **特殊性**: 这个没有常规API，主要依赖WebSearch工具
- **搜索方式**: 通过WebSearch工具获取实时网络数据
- **实施**: 在Claude Code环境中使用WebSearch工具

## ⚠️ 关键原则（每次修改代码都要自查）

### 1. 不要硬编码具体产品名
- ❌ 错误: `search_queries = ["ClawdBot", "Cursor AI", "v0.dev"]`
- ✅ 正确: `search_queries = ["Hacker News front page", "GitHub Trending AI", "most popular AI tools"]`

### 2. 让每个模型自己搜索和分析
- ❌ 错误: 我手动搜索然后手动整理数据
- ✅ 正确: 程序调用API，让模型自己分析并输出热点

### 3. 用ClawdBot作为验证点，不硬编码
- ❌ 错误: 在prompt中硬编码"ClawdBot"
- ✅ 正确: 搜索后检查是否包含ClawdBot，如果没有则调整搜索策略

### 4. 标注实时搜索状态
- 在报告中，实现了实时网络搜索的模型标注"（实时）"
- 例如: "GLM-4.6 (智谱AI)（实时）"

### 5. 通用发现模式
- 使用能发现热门产品和技术的通用查询
- 覆盖开发者社区（Hacker News, Reddit, GitHub）
- 不依赖具体产品名

## 🔄 程序流程

```
开始
  ↓
为每个模型定义搜索策略（通用查询，不硬编码）
  ↓
调用模型API
  ↓
模型基于其知识/搜索能力分析2026年AI趋势
  ↓
模型输出Top 5热点（每个150-200字，带来源链接）
  ↓
验证：是否找到ClawdBot？（作为测试点）
  ├─ 是 → 搜索策略有效 ✅
  └─ 否 → 调整搜索策略，重新搜索
  ↓
汇总所有模型的热点
  ↓
生成HTML报告
  ↓
标注实现实时搜索的模型（实时）
  ↓
完成
```

## 🎨 报告格式

每个模型的热点应包含：
1. **模型名称**（带"实时"标注）
2. **关注点**
3. **Top 5热点**
   - 热点标题
   - 深入分析（150-200字）
   - 权威来源链接（2-3个）
   - 关键指标

## 🚫 常见错误（避免）

1. **手动搜索后整理数据** ❌
   - 这是我在刚才做的事，不符合要求

2. **硬编码ClawdBot** ❌
   - 不要在搜索查询或prompt中硬编码具体产品名

3. **基于预定义数据生成报告** ❌
   - 必须通过API调用让模型自己分析

4. **只使用一个模型** ❌
   - 必须调用所有5个模型

## ✅ 正确实施方式

```python
# 伪代码示例
for model in [GLM_MODEL, CLAUDE_MODEL, CHATGPT_MODEL, GEMINI_MODEL]:
    # 调用模型API
    response = model_api.call(
        prompt="基于2026年的AI趋势，分析Top 5热点...",
        search_context=general_search_queries  # 通用查询，不硬编码
    )

    # 获取模型分析的热点
    hotspots = response.hotspots

    # 验证是否包含ClawdBot（作为测试点）
    if not contains_clawdbot(hotspots):
        adjust_search_strategy()

    all_model_results.append({
        "model": model.name,
        "hotspots": hotspots,
        "realtime_search": True  # 标注实时搜索
    })
```
