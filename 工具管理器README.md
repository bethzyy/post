# Post工具管理器 v2.0

一个可视化的Web应用，用于管理和运行post目录下的所有Python工具。

**2026-01-28更新**: 所有文件已按功能分类整理到子目录中！

## 功能特性

- ✅ **可视化界面**: 美观的Web界面，按类别展示所有工具
- ✅ **一键运行**: 点击按钮即可运行任何工具
- ✅ **实时状态**: 查看工具运行状态和输出
- ✅ **详细描述**: 每个工具都有清晰的功能介绍
- ✅ **分类管理**: 工具按功能分类到子目录，易于查找
- ✅ **目录结构优化**: 108个文件整理到5个功能分类中

## 快速开始

### 方法1: 使用启动脚本（推荐）

双击运行 `启动工具管理器.bat`

脚本会自动：
1. 检查并安装Flask（如果需要）
2. 启动Web服务器
3. 打开浏览器访问 http://localhost:5000

### 方法2: 手动启动

```bash
# 1. 安装依赖
pip install flask

# 2. 启动服务器
python tool_manager.py

# 3. 浏览器访问
# http://localhost:5000
```

## 使用说明

### 运行工具

1. 在Web界面中找到想要运行的工具
2. 点击"▶ 运行"按钮
3. 等待工具运行完成
4. 在弹出窗口中查看输出结果

### 查看工具详情

每个工具卡片显示：
- **工具名称和描述**: 清晰说明工具功能
- **文件名**: Python脚本文件名
- **最后修改时间**: 文件最后修改日期
- **文件大小**: 文件大小（KB）

### 运行状态

- 🔄 **运行中**: 工具正在执行（绿色）
- ✓ **完成**: 工具成功完成（青色）
- ✗ **失败**: 工具执行失败（红色）

## 技术架构

### 后端
- **Flask**: 轻量级Web框架
- **Python 3.8+**: 运行环境
- **Subprocess**: 进程管理

### 前端
- **HTML5**: 页面结构
- **CSS3**: 渐变背景和动画
- **JavaScript**: 异步API调用和状态更新

### API接口

#### GET `/`
返回主页面

#### GET `/api/tools`
获取所有工具列表

#### POST `/api/run`
运行指定工具
```json
{
  "filename": "multi_model_api_call.py"
}
```

#### GET `/api/status/<process_id>`
获取运行状态
```json
{
  "success": true,
  "filename": "multi_model_api_call.py",
  "status": "running",
  "elapsed_time": 10.5,
  "output": "...",
  "returncode": 0
}
```

#### POST `/api/stop`
停止运行中的工具
```json
{
  "process_id": "multi_model_api_call.py_1234567890"
}
```

## 添加新工具

### 方法1: 添加到现有分类

编辑 `tool_manager.py`，在 `TOOL_CATEGORIES` 中添加：

```python
TOOL_CATEGORIES = {
    "AI趋势分析": [
        "your_new_tool.py",  # 添加这里
        # ... 其他工具
    ]
}
```

### 方法2: 添加新分类

```python
TOOL_CATEGORIES = {
    "你的新分类": [
        "tool1.py",
        "tool2.py"
    ],
    # ... 其他分类
}
```

### 方法3: 添加工具描述

编辑 `tool_descriptions.json`：

```json
{
  "你的新分类": {
    "your_new_tool.py": "你的工具详细描述"
  }
}
```

## 注意事项

1. **Python环境**: 确保Python 3.8+已安装
2. **依赖包**: 某些工具需要特定的依赖包（如Flask、ZhipuAI SDK等）
3. **API密钥**: 某些工具需要配置API密钥（如ZHIPU_API_KEY）
4. **网络连接**: 部分工具需要网络连接才能正常工作

## 故障排除

### 问题1: 端口已被占用

```
Address already in use
```

**解决方案**: 修改 `tool_manager.py` 中的端口号

```python
app.run(host='0.0.0.0', port=5001, debug=True)  # 改为5001
```

### 问题2: Flask未安装

```
ModuleNotFoundError: No module named 'flask'
```

**解决方案**: 运行 `pip install flask`

### 问题3: 工具运行失败

**可能原因**:
- 缺少依赖包
- API密钥未配置
- 网络连接问题

**解决方案**: 查看工具的具体错误信息
